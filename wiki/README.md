# Wiki 说明

这是一个 **LLM Wiki** —— 由 Claude 维护的个人知识库。

## 📖 什么是 LLM Wiki？

不同于传统的 RAG（检索增强生成）系统，LLM Wiki 是一个**持久的、不断演化的知识库**：

- 🔄 **增量构建** - 每当你添加新源时，不只是索引，而是读取、提取、整合
- 🔗 **交叉引用** - 页面之间的连接已经建立，发现关系的工作提前做好了
- 📈 **累积价值** - 知识每次查询都在积累，不是每次都从头开始

## 📁 结构

```
.
├── index.md               ← 👈 从这里开始（主目录）
├── log.md                 ← 操作日志
├── entities/              ← 人物、组织、事件
├── concepts/              ← 理论、原则、框架
├── sources/               ← 源文档摘要
└── analyses/              ← 生成的分析
```

## 🚀 快速开始

### 1. 导入新源
```
你：我有一篇关于 X 的文章，放在 raw/articles/ 里了

我：读取 → 生成摘要 → 建立链接 → 更新索引
```

### 2. 查询知识库
```
你：这个知识库里有什么关于 Y 的内容？

我：搜索 index.md → 读取相关页面 → 综合回答 → 可能生成新分析
```

### 3. 浏览和探索
在 Obsidian 中：
- **Graph View** - 看知识如何连接
- **Search** - 快速找到相关页面
- **Backlinks** - 看什么页面引用了当前页

## 📝 基本约定

- 所有文件都是 Markdown
- 使用 `[[双括号]]` 创建内部链接
- Front Matter 包含元数据（标签、关系、状态）
- 双向链接（A→B 时，B 也链接回 A）

## 💡 何时最有用？

LLM Wiki 适合：
- 📚 深度研究某个领域（积累周期内的所有知识）
- 📖 阅读书籍时逐章构建伴随 wiki
- 🔬 项目和学习的长期知识积累
- 🧠 个人知识管理和自我反思
