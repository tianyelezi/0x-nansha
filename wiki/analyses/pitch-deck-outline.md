## 详细脚本（每页）

### 📄 页 01 ─ 封面

```
> ▸ 南沙产业知识图谱平台
> "2 小时，编译南沙产业知识图谱"

- **视觉**：满屏黑底 + 大字号 ASCII `NANSHA` banner + 霓虹青/品红渐变 + Clawtime 龙虾图腾右下角
- **口播**：「大家好。接下来 6 分钟，我演示一件事 — 一个人 + 一个 Claude，在 2 小时之内，能不能把南沙整个 8+2+3 产业体系，编译成一张可被任何人浏览、可被 AI 直接调度的知识图谱。」

### 📄 页 02 ─ PROBLEM

- **主标**：南沙招商的"产业失焦"难题
- **三点痛**：
  1. **13 条赛道**散落在不同政策文件 / 政府部门 / 招商手册 — 没人能一眼看全
  2. **企业**常常同时属于多条赛道（如 SiC 半导体 = IC + 汽车 + 新能源），但官网按"一企一档"陈列
  3. **AI Agent 时代**：传统 PDF 招商手册无法被大模型直接读取调用
- **视觉**：3 张分裂的截图（南沙官网 / 政策 PDF / Excel 招商表）
- **口播**：「南沙不缺资料，缺的是结构。13 条产业赛道散在政策、官网、招商手册里 — 人记不住，AI 也读不动。」

### 📄 页 03 ─ MISSION

- **主标**：命题边界
- **三个约束**：
  - ⏱ **2 小时硬时限**
  - 🤖 **LLM × 1 人** ─ 没有团队、没有设计师
  - 🔗 **产出可链接 wiki**（不是 PDF、不是 PPT）
- **视觉**：倒计时图标 + Clawtime 大赛 logo
- **口播**：「Clawtime 大赛的命题特别狠 — 2 小时、1 人 + AI、必须产出可链接的 wiki。」
- **链接**：[[0x-nansha-brief]]

### 📄 页 04 ─ METHOD

- **主标**：Karpathy LLM Wiki 范式 — 单一数据源，三层架构
- **架构图**：

  ```
  raw/      ← 原始资料（人喂）
     ↓
  wiki/     ← 知识库（AI 编译，markdown + `[[wikilink]]`）
     ↓
  site/     ← Quartz 静态站（自动产出 / 全文搜索 / 图谱视图）
     ↓
  GitHub Pages / Obsidian 同时挂载
  ```

- **视觉**：纵向三层架构图，每层一种霓虹色
- **口播**：「我用的是 Karpathy 提出的 LLM Wiki 范式。一份 markdown 数据源，三种形态消费 — 浏览器看、Obsidian 看、AI Agent 读。」
